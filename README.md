## Configuration

The flight route system can be configured mainly through two parts:

1. **Flight Profile**  
   Defines the aircraft performance limits and safety values.

2. **Flight Pathfinder Actor**  
   Defines how the 3D route search is built, how the aircraft can move through the voxel grid, and how debug output is displayed.

---

## Flight Profile

The `FlightProfile` is a Data Asset that stores the aircraft-specific values used by the pathfinding system.  
Different aircraft can use different profiles, which makes it possible to test route planning with different performance limits.

### Aircraft Settings

| Setting | Description | Effect |
|---|---|---|
| `AircraftName` | Display name of the aircraft profile. | Used to identify the profile in the editor. |
| `MaxAltitudeMetersASL` | Maximum allowed altitude above sea level. | The pathfinder will not allow route points above this altitude. |
| `CruiseSpeedMetersPerSecond` | Normal cruise speed of the aircraft. | Used for flight-related calculations such as time estimates. |
| `MaxClimbRateMetersPerSecond` | Maximum vertical climb speed. | Limits how steeply the aircraft is allowed to climb. |
| `MaxDescentRateMetersPerSecond` | Maximum vertical descent speed. | Limits how steeply the aircraft is allowed to descend. |
| `RangeMeters` | Maximum flight distance of the aircraft. | Can be used to check whether a route is within the aircraft's range. |
| `StallSpeedMetersPerSecond` | Minimum safe airspeed. | Represents the lowest safe flight speed of the aircraft. |
| `MinimumTurnRadiusMeters` | Smallest safe turn radius. | Used to represent how tightly the aircraft can turn. Larger values create smoother, less aggressive routes. |

### Safety Settings

| Setting | Description | Effect |
|---|---|---|
| `MinimumTerrainClearanceMeters` | Minimum required safety distance above terrain. | The pathfinder checks that the aircraft remains above the terrain by at least this distance. Higher values create safer but potentially longer or higher routes. |

## Flight Pathfinder Actor

The `FlightPathfinderActor` performs the aircraft-aware A* route search.  
It uses a 3D voxel search space and includes the aircraft heading direction as part of the search state. This allows the system to create routes that better respect aircraft movement instead of only searching from point to point.

---

## Required References

Before a route can be calculated, the pathfinder needs several references.

| Setting | Description | Effect |
|---|---|---|
| `GridBaker` | Reference to the baked voxel grid actor. | Provides the base grid structure used for pathfinding. |
| `HeightCache` | Reference to the baked terrain height data. | Used to check terrain height and terrain clearance. |
| `FlightProfile` | Reference to the aircraft profile. | Provides aircraft limits such as altitude, climb rate, descent rate, and safety clearance. |
| `StartActor` | Actor used as the route start position. | The route search starts near this actor. |
| `GoalActor` | Actor used as the route target position. | The route search tries to reach this actor. |
| `InfluenceZones` | List of influence zone actors. | Can define blocked areas or areas with additional traversal cost. |

---

## Search Space Settings

These settings control the vertical search space used by the pathfinder.

| Setting | Default | Description | Effect |
|---|---:|---|---|
| `VoxelSizeZMeters` | `25.0` | Height of one vertical voxel layer. | Smaller values create more detailed altitude steps but increase search cost. Larger values improve performance but reduce vertical precision. |
| `ExtraBottomLayers` | `0` | Additional layers below the relevant terrain or route height. | Allows the search space to extend further downward. Usually kept low for flight routes. |
| `ExtraTopLayers` | `8` | Additional layers above the relevant terrain or route height. | Allows the aircraft to climb higher during pathfinding. Higher values give the pathfinder more vertical freedom but increase calculation time. |

### Visible Search Space Values

These values are generated by the system and are mainly used for checking/debugging.

| Setting | Description |
|---|---|
| `SearchMinWorld` | Minimum world-space bounds of the generated search space. |
| `SearchMaxWorld` | Maximum world-space bounds of the generated search space. |
| `ZLayerCount` | Number of vertical layers in the generated search space. |

---

## Flight Model Settings

These settings control how the aircraft can move through the voxel grid.

| Setting | Default | Description | Effect |
|---|---:|---|---|
| `HeadingBucketCount` | `16` | Number of possible flight directions. | Higher values allow more precise direction changes but increase the search state count. Lower values improve performance but create more angular routes. |
| `MaxHeadingChangePerStep` | `1` | Maximum allowed heading change per search step. | Lower values force smoother turns. Higher values allow sharper direction changes. |
| `bAllowClimbStep` | `true` | Allows movement to one layer above the current layer. | If disabled, the pathfinder cannot climb during the search. |
| `bAllowDescentStep` | `true` | Allows movement to one layer below the current layer. | If disabled, the pathfinder cannot descend during the search. |

---

## Influence Zones

Influence zones can affect the route search in two different ways:

| Zone Type | Effect |
|---|---|
| Hard block zone | The route is not allowed to pass through this area. |
| Soft cost zone | The route is still allowed to pass through the area, but the movement cost is increased. |

This makes it possible to model areas that should be avoided, such as restricted airspace, dangerous terrain areas, or regions where flight should be discouraged but not fully blocked.

---

## Debug Settings

The pathfinder includes several debug options for visualizing the route search.

| Setting | Default | Description | Effect |
|---|---:|---|---|
| `bDrawVisitedStates` | `false` | Draws checked A* states. | Useful for debugging, but can reduce editor performance when many states are visited. |
| `bAutoDrawPathAfterSearch` | `true` | Automatically draws the route after a successful search. | Makes the resulting path visible immediately after calculation. |
| `DebugDrawLifetime` | `20.0` | Lifetime of temporary debug shapes in seconds. | Controls how long debug visuals remain visible. |
| `DebugLineThickness` | `6.0` | Thickness of the drawn route line. | Higher values make the route easier to see. |
| `DebugVisitedPointSize` | `16.0` | Size of the debug points for visited states. | Higher values make visited states more visible. |

---

## Result Data

After a successful route search, the pathfinder stores the generated route.

| Setting | Description |
|---|---|
| `CurrentRouteWorldPoints` | Final route as world-space positions. |

This array contains the calculated route points and can be used for visualization, analysis, or later flight logic.

---

## Editor Functions

The pathfinder provides several editor-callable functions.

| Function | Description |
|---|---|
| `BuildSearchSpace()` | Builds the 3D search space from the terrain cache. This should be called before searching for a route. |
| `FindFlightRoute()` | Runs the aircraft-aware A* route search. |
| `DebugDrawCurrentRoute()` | Draws the current route in the editor viewport. |
| `ClearCurrentRoute()` | Clears the current route result and debug drawings. |

## Recommended Workflow

1. Create or select a `FlightProfile` Data Asset.
2. Set the aircraft performance values, such as maximum altitude, climb rate, descent rate, terrain clearance, and turn radius.
3. Place a `FlightPathfinderActor` in the level.
4. Assign the required references:
   - `GridBaker`
   - `HeightCache`
   - `FlightProfile`
   - `StartActor`
   - `GoalActor`
5. Adjust the search space settings if needed.
6. Adjust the flight model settings depending on the desired route behavior.
7. Call `BuildSearchSpace()`.
8. Call `FindFlightRoute()`.
9. Use the debug settings to inspect the generated route.

---

## Performance Notes

Some settings can strongly affect performance:

| Setting | Performance Impact |
|---|---|
| Smaller `VoxelSizeZMeters` | Increases vertical precision but creates more search layers. |
| Higher `ExtraTopLayers` | Gives the pathfinder more altitude options but increases the search space. |
| Higher `HeadingBucketCount` | Allows smoother direction handling but increases the number of possible states. |
| Higher `MaxHeadingChangePerStep` | Creates more neighbor options per state, which can increase search time. |
| `bDrawVisitedStates = true` | Can become expensive when many states are visited. |

For faster searches, use a larger vertical voxel size, fewer extra layers, fewer heading buckets, and disable visited-state debug drawing.

For more detailed and realistic routes, use a smaller vertical voxel size, more heading buckets, and stricter aircraft movement limits.
